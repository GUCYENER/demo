# B1 — Saved report rerun fix

## 1. Tetikleyici (Why)
Kullanıcı kayıtlı raporu "Çalıştır" butonu ile yeniden tetiklediğinde 500 dönüyor:
> ValueError: Desteklenmeyen veritabanı tipi: db_type

## 2. Kök Sebep (kanıtlanmış)
**Dosya**: `app/api/routes/db_smart_api.py:903-911` — `_load_source` fonksiyonu kayıtlı rapor JSON'undan source yüklerken `dialect` değişkenini normalize ediyor, ama `source_dict["db_type"]` literal `"db_type"` string'i kalıyor. Downstream `ds_learning_service._get_db_connector(source_dict["db_type"])` literali DB type sanıyor → `app/services/ds_learning_service.py:80` raise.

## 3. Hedef (What)

### 3.1 Backend fix
- `app/api/routes/db_smart_api.py` `_load_source`: normalize sonrası
  ```python
  source_dict["db_type"] = dialect
  ```
  satırını ekle (normalize sonrası, return öncesi).

### 3.2 Defensive guard
- `app/services/ds_learning_service.py` `_get_db_connector`:
  - `db_type == "db_type"` veya `db_type in (None, "", "db_type")` durumunda daha açıklayıcı `ValueError("Geçersiz/normalleştirilmemiş db_type değeri: %r — _load_source çağrısı kontrol edin" % db_type)` raise et.

### 3.3 Retro backfill — migration 047b
- `migrations/047b_v3370_saved_reports_db_type_backfill.py`:
  - `saved_reports.source_json` içinde `source.db_type == 'db_type'` olan kayıtlar için `connections` tablosundan engine info join edilerek `db_type` doğru değere update edilir.
  - **Idempotent**: 2. çalıştırmada hiçbir satıra dokunmaz.
  - **Dry-run flag**: `--dry-run` ile sadece etkilenecek kayıt sayısını raporla, UPDATE yapma.
