## 📜 CONTEXT (Neden?)

Mevcut VYRA "Veritabanında Ara" akışında 3 boşluk var:

1. **DB tarafında öğrenme yok** — `learned_answers` sadece RAG için; başarılı SQL sorguları cache'lenmiyor → her sorguda LLM tam pipeline çalışıyor (~3-5sn).
2. **FK ilişkileri kullanılmıyor** — `ds_db_relationships` keşfediliyor ama "örnek JOIN sorgusu üret-çalıştır-öğret" döngüsü yok.
3. **Few-shot havuzu manuel** — `few_shot_examples` admin tarafından dolduruluyor; kullanıcının başarılı sorguları otomatik örneğe dönüşmüyor.

**Hedef:** Self-improving DB query loop. Her FK için **2 örnek** öğren (duble yok), başarılı sorguları **otomatik cache**'le, **AST shortcut** ile basit sorularda LLM'i atla.

**Kullanıcı şartları:**
- ✅ N FK ilişkisinin her biri için tam 2 örnek yeter
- ✅ Dedupe zorunlu — 3 katman: SQL hash + question embedding + schema fingerprint
- ✅ Tüm fazlar uygulanacak (G1-G9)
