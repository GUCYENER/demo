# v3.29.9 — FK Inference Layer (convention + sample validation, multi-dialect)

**Branch:** hira • **Tarih:** 2026-05-19 • **Sürüm:** v3.29.9
**Dialect kapsamı:** PostgreSQL, Oracle, MSSQL, MySQL (hepsi v3.29.9)
**v3.29.8 entegrasyon:** signal_weight_overrides, signal_weight_analyzer, signal_breakdown event, agentic_observability

---

## Context

OnedeskTest gibi enterprise DB'lerde declared FK sayısı çok düşük (29 / 2138 tablo). DB doğrulanmıştır: `pg_constraint WHERE contype='f'` toplamı **29**. ORM/app-layer FK enforcement (ELYSION pattern) sebebiyle FK'lar DB'ye konmamış. VYRA mevcut [ds_learning_service.py:391-470](app/services/ds_learning_service.py#L391-L470) sadece declared FK'ları okuyor — convention/naming bazlı çıkarım yok. Bu, multi_signal_rank'in `fk_centrality`, multi-hop path resolver (Faz 6 G3), Query Builder v2 (Faz 6 G6) gibi tüm downstream katmanların temelini zayıflatıyor.

Hedef: **Obje tespiti** adımından sonra otomatik (veya admin tetikli) çalışan bir **inference servisi** her tablo için olası FK'ları çıkarır, sample-data ile doğrular, `ds_db_relationships`'e `is_inferred=true, admin_verified=false` olarak yazar. Admin UI'dan onay/red akışı.

## Tasarım

### 1) Migration `031_v3299_fk_inference_metadata.py`

`ds_db_relationships` tablosuna kolon ekle:
- `is_inferred BOOL NOT NULL DEFAULT FALSE` — declared vs inferred ayrımı
- `confidence FLOAT` — 0.0–1.0
- `inference_method VARCHAR(32)` — 'naming', 'naming+type', 'naming+type+sample'
- `evidence_json JSONB` — `{naming_pattern, type_match, sample_coverage, sample_size, ...}`
- `admin_verified BOOL NOT NULL DEFAULT FALSE`
- `verified_by INT REFERENCES users(id) ON DELETE SET NULL`
- `verified_at TIMESTAMPTZ`
- `rejected_at TIMESTAMPTZ` — admin "reddetti" akışı için (NULL = aktif)

**Backfill:** mevcut FK'lara `is_inferred=false, admin_verified=true` SET. Idempotent (IF NOT EXISTS kolonlar + UPDATE WHERE is_inferred IS NULL).

Index: `idx_dsdrel_inferred ON ds_db_relationships(source_id, is_inferred, admin_verified)`.

### 2) Servis `app/services/db_learning/fk_inference_service.py`

**Public API:**
```python
infer_fks_for_source(cur, source_id: int, *,
    sample_validate: bool = True,
    sample_rows: int = 200,
    min_confidence: float = 0.7,
    dialect: str = 'postgresql',
) -> Dict[str, int]
```

Dönüş: `{candidates, persisted, skipped_low_conf, skipped_existing}`.

**Adım 1 — Schema snapshot (1 sorgu):**
- Per-tablo PK kolonu (`pg_constraint contype='p' AND array_length(conkey,1)=1` → tek-kolon PK)
- Her tablonun kolon listesi + tipi (`ds_db_objects`'tan veya canlı `information_schema.columns`'tan)
- In-memory dict: `pks: {(schema, table): {col_name, col_type}}`

**Adım 2 — Naming match (Python, no DB):**
- Her tablonun her kolonu için pattern test:
  - `<x>_id`, `<x>id`, `<x>_no`, `id_<x>` → `x` = table_root candidate
  - Türkçe ek: `<x>_kimlik`, `<x>_kodu` (low priority)
- `x` → muhtemel hedef tablo: aday set:
  - Tam eşleşme: `tables[x]`
  - Plural→singular: `users`→`user`, `companies`→`company` (regex)
  - Singular→plural: `user`→`users`
  - Schema-cross: sadece aynı schema (cross-schema FK nadir; v3.29.10'a bırak)
- Skor temeli: **0.6** (naming).

**Adım 3 — Type check (in-memory):**
- from_column.type ≈ to_table.pk.type (INT-INT, BIGINT-BIGINT, UUID-UUID, VARCHAR-VARCHAR)
- Eşleşirse: **+0.15** → 0.75
- INT↔BIGINT uyumlu kabul (cross-cast bilinen pattern).

**Adım 4 — Sample validation (opsiyonel, DB heavy):**
- `sample_validate=True` ve naming+type ≥ 0.7 olan adaylar için:
  - `SELECT DISTINCT from_col FROM from_t WHERE from_col IS NOT NULL LIMIT 200`
  - `SELECT COUNT(*) FROM to_t WHERE to_pk IN (sample)` → coverage_ratio
  - coverage ≥ 0.9 → **+0.25** (toplam ~1.0)
  - coverage 0.5–0.9 → +0.10
  - coverage < 0.5 → -0.30 (false positive sinyali; eleme)
- Timeout per query: 2 saniye (statement_timeout). Hata → skip, log.

**Adım 5 — Persist:**
- `confidence >= min_confidence` (default 0.7) olanlar `ds_db_relationships`'e UPSERT:
  - `ON CONFLICT (source_id, from_schema, from_table, from_column, to_schema, to_table, to_column) DO NOTHING`
  - `is_inferred=TRUE`, `admin_verified=FALSE`, `evidence_json` doldurulur.
- Declared FK'larla çakışan adaylar atlanır (zaten `admin_verified=TRUE` rakam görür).

**Performance budget (2138 tablo, ~38K kolon):**
- Schema snapshot: 1 query, ~1 sn
- Naming match (Python): O(K) ≈ 38K iter, ~2 sn
- Type check: in-memory, ~0
- **No-sample mode:** ~5 sn total
- **Sample mode:** her aday için 2 sorgu × ~2000 aday × 200ms = ~13 dk; 5 paralel worker → ~3 dk
- Default: `sample_validate=True` ama "Hızlı tarama" toggle (sample_validate=False) UI'da seçenek olarak.

**Dialect — adapter pattern (v3.29.9 = TÜMÜ):**

`fk_inference_service.py` içinde dialect-agnostic core + per-dialect adapter:

```python
class FKInferenceDialect(Protocol):
    name: str
    def fetch_pk_snapshot(self, cur, source_filter) -> Dict[Tuple[str,str], Dict]
    def fetch_columns_snapshot(self, cur, source_filter) -> Dict[Tuple[str,str], List[Dict]]
    def fetch_distinct_sample(self, cur, schema, table, col, limit) -> List[Any]
    def count_pk_overlap(self, cur, schema, table, pk_col, values) -> int
    def quote_ident(self, name) -> str
    def statement_timeout_ms(self, cur, ms: int) -> None
```

**4 implementasyon:**

| Dialect | PK snapshot | Sample LIMIT | Quote | statement_timeout |
|---|---|---|---|---|
| **PostgreSQL** | `pg_constraint contype='p' AND array_length(conkey,1)=1` JOIN `pg_attribute` | `LIMIT N` | `"name"` | `SET LOCAL statement_timeout = N` |
| **Oracle** | `all_constraints WHERE constraint_type='P'` JOIN `all_cons_columns` (single col); owner = upper-case normalize | `FETCH FIRST N ROWS ONLY` (12c+) veya `ROWNUM <= N` (11g) | `"NAME"` | `BEGIN DBMS_RESOURCE_MANAGER.SWITCH_PLAN... END;` yok → fallback `cx_Oracle.callTimeout` veya skip-on-slow |
| **MSSQL** | `sys.key_constraints type='PK'` JOIN `sys.index_columns` (single col) | `SELECT TOP N` | `[name]` | `SET LOCK_TIMEOUT N` (read-only sorgular için yeterli) |
| **MySQL** | `INFORMATION_SCHEMA.TABLE_CONSTRAINTS CONSTRAINT_TYPE='PRIMARY KEY'` JOIN `KEY_COLUMN_USAGE` (COUNT=1) | `LIMIT N` | `` `name` `` | `SET SESSION MAX_EXECUTION_TIME=N` (8.0+) |

**Identifier case-normalization:**
- Oracle: tüm identifiers UPPERCASE depolanır → karşılaştırma `.upper()`
- PG: lowercase default (case-sensitive olanlar quoted) → karşılaştırma `.lower()` (zaten mevcut [ds_learning_service.py](app/services/ds_learning_service.py) pattern)
- MSSQL: case-insensitive collation default → `.lower()` safe
- MySQL: lower_case_table_names config; default Linux'ta case-sensitive → `.lower()` yumuşak normalize, eşleşmezse case-sensitive retry

**Dialect detection:** mevcut `ds_data_sources.dialect` kolonundan; eski kaynaklar `'postgresql'` default.

**Naming patterns dialect-aware:**
- Tüm dialect'lerde ortak: `<x>_id`, `<x>id`, `id_<x>`
- Oracle/Enterprise: `<x>_no`, `<x>_kod`, `<x>_kodu` (TR), `<x>_code`, `<x>_num`
- ELYSION/.NET: `<x>Id` (camelCase), `f_<x>` (foreign prefix)
- Pattern set per-dialect `NAMING_PATTERNS[dialect]` (mevcut yine module-level constant, dialect override edebilir)

**Sample query güvenliği:**
- Tüm dialect'lerde `IN (%s, %s, ...)` parametrize (asla string concat değil)
- Identifier'lar adapter.quote_ident'ten geçer (regex pre-check: `^[A-Za-z0-9_]+$`, başkası → skip + log)
- Mevcut Faz 6 G3 [query_builder_api.py](app/api/routes/query_builder_api.py)'deki whitelist regex pattern'i reuse

### 6) v3.29.8 Signal Weight Tuner entegrasyonu (tam senkron)

**6a) `fk_centrality` semantik değişikliği:**
[multi_signal_rank.py](app/services/pipeline/nodes/multi_signal_rank.py)'deki `fk_centrality_score` hesabı şu kuralla genişletilir (declared+inferred birleşik graph):
- `admin_verified=TRUE AND rejected_at IS NULL` → full weight (1.0) — declared FK'lar default burada
- `is_inferred=TRUE AND admin_verified=FALSE AND rejected_at IS NULL` → **0.5 × confidence**
- `rejected_at IS NOT NULL` → 0 (sayım dışı)

Centrality counter artık `sum(weight) / max_centrality` (önceden tam sayım); `max_centrality` 10 (mevcut) korunur ama doygunluk daha yumuşak.
