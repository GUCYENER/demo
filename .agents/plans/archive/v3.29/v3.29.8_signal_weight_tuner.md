## Pearson Korelasyon Yaklaşımı (L2)

Her sinyal için:
1. Son 7 gün `signal_breakdown` event'lerini topla (top-1 candidate'ın sinyal skoru)
2. Aynı run_id'nin `pipeline_end` outcome'ını eşle (1=ok, 0=error/retry>0)
3. Pearson r(signal_score, outcome) hesapla
4. n < MIN_SAMPLE_SIZE (50) → öneri üretme

Önerilen weight formülü:
```python
suggested = current * (1 + lambda_ * r)
suggested = clip(suggested, current * 0.5, current * 2.0)  # max %100 drift
normalize_all_weights_to_sum_1()
confidence = abs(r) * sqrt(n / (n + 30))  # Bayesian shrinkage
```

**Güvenlik:** lambda_ = 0.3 (yumuşak ayarlama). Confidence < 0.5 ise admin_verified zorunlu (auto-apply yok).

## Risk Özeti

| Risk | Olasılık | Etki | Mitigasyon |
|------|----------|------|------------|
| Pearson korelasyon yanıltıcı (confounder) | Yüksek | Orta | n<50 skip, |r|<0.1 skip, admin onayı zorunlu |
| Cache stale weight | Orta | Düşük | 60s TTL + manual invalidate endpoint |
| Multi-tenant weight sızıntısı | Düşük | Yüksek | company_id ile sıkı RLS + UNIQUE constraint |
| Scheduler thread'de DB connection leak | Orta | Orta | with-block + connection pool |
| Storage bloat (signal_breakdown her run = 1 event) | Orta | Düşük | 30 gün retention (pipeline_events auto-purge job) |
