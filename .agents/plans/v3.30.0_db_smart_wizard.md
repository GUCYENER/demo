## 1. Context (Neden bu değişiklik?)

Kullanıcı, mevcut "Veritabanında Ara" (chat tarzı tek atımlık doğal dil sorgu)
deneyiminin ötesinde, **wizard tarzı bir SQL copilot** — "Akıllı Veri Keşfi"
modülü istiyor. db_smart_prompts.md dosyasında ayrıntılı vizyon belgesi
(Prompt A→J) verilmiş; **bu plan o vizyonun TAM kapsamını** hayata geçirir.
MVP kısaltması yok — Prompt A (mimari), B (journey map), C (veri modeli),
D (state machine), E (metrik kütüphanesi — 5 domain × 8-12 metrik),
F (query generation + AST + dialect rendering + optimizasyon),
G (UX/CX — a11y, mikro-etkileşim, onboarding, power user shortcuts),
H (akıllı rapor önerisi — data shape analysis + rule engine + insights),
I (learning loop — event taxonomy + feature store + CatBoost retrain + bandit
exploration + privacy), J (yol haritası — fazlar) tamamı uygulanacak.

**İlke:** Mevcut text-to-SQL/RAG/öğrenme/multi-DB altyapısının üzerine paralel
ikinci giriş; kb/db/llm akışları **bozulmadan** dördüncü kart eklenir.
Türkçe iş terminolojisi (business_glossary_v2, ds_column_enrichments,
ds_code_values) ile **iş dilinden SQL'e** çeviri. 4 dialect
(PostgreSQL/Oracle/MSSQL/MySQL) + multi-tenant izolasyon (RLS).

**Kart adı:** "Akıllı Veri Keşfi" · **Açıklama:** "Adım adım yönlendirmeli veri keşfi"
**Kart rengi:** `var(--accent)` (turkuaz) — kb=mavi, db=yeşil, **akıllı_keşif=turkuaz**, llm=mor

**Hedef versiyon:** v3.30.0 (yeni minör modül).
Önkoşul: ✅ v3.29.11 versiyon senkronizasyon commit'i atıldı (dfda5a1).

| Metric library yetersizliği (kullanıcı bulamıyor) | Orta | Orta | 5 domain × 8-12 metrik (toplam ~50) + custom metric path + community templates Faz 4. |
| CatBoost cold start kötü performans | Yüksek | Orta | Sentetik veri baseline + tenant clustering + fast-learn mode. |
| LLM hallucination custom metric / narrative | Orta | Yüksek | Schema_context sınırlama + identifier whitelist + rationality guard (narrative). |
| Migration head conflict (031 sonrası 032-034) | Düşük | Orta | down_revision explicit chain, alembic test. |

| Streaming (>100K satır) connection pool drain | Düşük | Orta | Ayrı connection pool (max 5), asyncio queue, max_lag 5sn cancel. |
| A/B testing yanlış metrik kayması | Düşük | Orta | Stat sig p<0.05, n>1000, otomatik rollback >5% drop. |
| Schedule (cron) sistem kaynağı sömürme | Düşük | Orta | Per-user concurrent run limit (max 3), resource quota. |
| PII sızıntısı (interactions/narrative) | Orta | Çok Yüksek | ds_column_enrichments.is_pii guard + masking + KVKK/GDPR retention. |
