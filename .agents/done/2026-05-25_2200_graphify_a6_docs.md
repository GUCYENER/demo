## `examples/usage.md`

3 senaryo, her biri tam copy-paste runnable:

1. **Initialize a new project**
   ```bash
   graphify init myproj --path /path/to/repo
   graphify mine --project-cwd /path/to/repo
   graphify status --project myproj
   ```
2. **Search**
   ```bash
   graphify search "auth middleware" --project myproj --mode hybrid --limit 5
   ```
3. **Python API**
   ```python
   from pathlib import Path
   from core.graphify import Graphify

   g = Graphify(Path.home() / ".graphify/instances/myproj.db")
   g.init()
   ent = g.add_entity("Refactor", "R001", properties={"priority": "high", "status": "open"})
   results = g.find(entity_type="Refactor", limit=10)
   for r in results:
       print(r["name"], r["properties"])
   ```

## `examples/basla_bitir_integration.md`

Plan §11 ozeti — BAŞLA + BİTİR workflow icine Graphify nasil sokulur, hangi tool ne zaman cagrilir.

```markdown
## BAŞLA (Session Startup)

1. mcp__graphify__warmup()
2. mcp__graphify__wakeup(project="<slug>")
3. If stale (last_indexed_commit != HEAD):
   mcp__graphify__mine(project="<slug>", since="auto")
4. Continue with services check + existing workflow

## BİTİR (Session Wrap-up)

1. Quality gates (existing)
2. mcp__graphify__add_decision(commit_msg, branch, council, refactor_ids, bug_ids)
3. git commit + push (existing)
4. mcp__graphify__mine (sync new commit)
5. Spot-check: mcp__graphify__search(<commit_sha>) — should find Commit entity
```
